# CreatePlan Tool Invocation Contract

> Loaded on-demand by `tl-agent-plan-create` when invoking the CreatePlan tool. See `../SKILL.md` for the parent skill.

The `CreatePlan` tool generates YAML frontmatter automatically from the `todos` argument and prepends it to the `plan` argument string. Therefore:

**The `plan` argument MUST begin with the H1 heading (`# Plan Title`) and MUST NOT contain a `---` frontmatter fence.**

The Technical Plan Template and Strategic Plan Template show the *destination file shape* — what the resulting `.plan.md` looks like on disk after the tool finishes. The frontmatter half of those templates (the `---\nname:\noverview:\ntodos:\n...\n---` block) is generated by the tool from the `todos` argument; you do not author it inside `plan`.

## Correct invocation

```
CreatePlan(
  name: "my-plan",
  overview: "One-sentence summary",
  todos: [{ id: "t1-1-1", content: "..." }, ...],
  plan: "# My Plan\n\n## Plan Metadata\n\n| Field | Value |\n|-------|-------|\n| Status | planned |\n..."
)
```

The `plan` argument starts at the H1.

## Incorrect invocation

```
CreatePlan(
  name: "my-plan",
  todos: [...],
  plan: "---\nname: my-plan\ntodos:\n  - id: t1-1-1\n...\n---\n\n# My Plan\n..."
)
```

The `plan` argument starts with `---`, duplicating the frontmatter the tool will prepend.

## Common failure

Two stacked `---` frontmatter blocks at the top of the plan file. Cursor's plan parser only recognizes the first block; the second block (and the rest of the body) becomes structurally orphaned, and downstream readers (the audit skill, the execute skill, the IDE plan view) cannot reliably parse the document. Fix is `StrReplace` to delete the duplicate fence and the lines between it and the H1.

> Decision: never paste a `---` fence into the `plan` argument — start it at the H1. The templates illustrate the file, not the call.
